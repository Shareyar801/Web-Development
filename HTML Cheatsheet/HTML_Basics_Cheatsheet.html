Introduction To HTML:

        <!DOCTYPE html>
        <html lang="en">

        <head>
                <meta charset="">
                <meta http-equiv="" content="">

                <title>Document

Any html document will have this syntax. First we have <head> tag which is basically a container for declaring elements. In other words <head> element is a container for metadata (data about data). Head may contain: <title>, <style>, <meta>, <link>, <script>, <base>.

01.The <title> element defines the title of the document. The title must be text-only.
02.The <style> is used to define style with html document and offers css within file.
03.The <link> element defines the relationship between the current document and an external resource. It is most often used to link to external style sheets.
04.The <base> element specifies the base URL and/or target for all relative URLs in a page.
05.The <body> element is main area where all the work is done.
06.The <meta> element is typically used to specify the character set, page description, keywords, author of the document, and viewport settings. The metadata will not be displayed on the page, but is used by browsers (how to display content or reload page), by search engines (keywords), and other web services.

<meta> Tags examples:
a.Use "<meta name="keywords" content="HTML, CSS">" to define keywords for search engines.
b.Use "<meta name="description" content="Free html tutorial">" to define a description of your web page.
c.Use "<meta name="author" content="SF">" to define the author of a page.
d.Use "<meta http-equiv="refresh" content="30">" to define duration of refresh document.
e.Use "<meta name="viewport" content="width=device-width, initial-scale=1.0">" to define viewport to make your website look good on all devices.

Block & Inline Elements:

01.Block Level:

A block-level element always starts on a new line, and the browsers automatically add some space (a margin) before and after the element. A block-level element always takes up the full width available (stretches out to the left and right as far as it can).
The <p> element defines a paragraph in an HTML document.
The <div> element defines a division or a section in an HTML document.
Other block Tags are: <address>, <article>, <blockquote>, <aside>, <footer>, <section>, <canvas>, <figure>, <main>, <tfoot>, <noscript>, <nav>, <fieldset>, <figcaption>, <video>

02.Inline Level:

An inline element does not start on a new line.An inline element only takes up as much width as necessary.
The <span> element does not make any visual change by itself. span is very similar to the div tag, but div is a block-level tag and span is an inline tag.
Other block Tags are: <a>, <abbr>, <acronym>, <big>, <cite>, <dfn>, <kbd>, <object>, <samp>, <time>, <var>, <label>, <select>, <script>, <textarea>

CLASS ID
The "class" assigned to an element has its name starts with "." followed by class name. In HTML, for an element, the ID name starts with the "#" symbol followed by a unique name assigned to it.
Multiple class selectors can be attached to an element. Only one ID selector can be attached to an element.
The class can be applied to multiple elements so it could be multiple times on a single page. ID is unique in a page and can only apply to at most one element

Responsive Web Design:

Responsive Web Design is about using HTML and CSS to automatically resize, hide, shrink, or enlarge, a website, to make it look good on all devices (desktops, tablets, and phones).A responsive web design will automatically adjust for different screen sizes and viewports. There are serval way through which we can achieve this.

  • Setting Viewport: Adding viewport property in meta of your page will give the browser instructions on how to control the page's dimensions and scaling.
    Example: <meta name="viewport" content="width=device-width, initial-scale=1.0">

  • Responsive Images: Responsive images are images that scale nicely to fit any browser size. Using width and max-width properties from CSS we can adjust and scale image up and down.
    Example: <img src="img_girl.jpg" style="width:100%;">

    Example: <img src="img_girl.jpg" style="max-width:100%;height:auto;">

  • Responsive Text Size: The text size can be set with a "vw" unit, which means the "viewport width". That way the text size will follow the size of the browser window.
    Example: <h1 style="font-size:10vw">Hello World</h1>

    Hello World!!!

  • Responsive Designing through Bootstrap: A popular CSS framework is Bootstrap. Bootstrap uses HTML, CSS and jQuery to make responsive web pages.

  3. <input> Element Attributes:

    It is used to specifies that an input field must be filled out in particular way before submitting the form.
    ATTRIBUTES: DESCRIPTION: EXAMPLE: RESULT:
    Value: The input value attribute specifies an initial value for an input field. value=example
    Readonly: The input readonly attribute specifies that an input field is read-only. A read-only input field cannot be modified (however, a user can tab to it, highlight it, and copy the text from it). readonly
    Step: The input step attribute specifies the legal number intervals for an input field. Example: if step="3", legal numbers could be -3, 0, 3, 6, etc. step="3"
    Disabled: The input disabled attribute specifies that an input field should be disabled. A disabled input field is unusable and un-clickable. disabled
    Maximum Length: The input maxlength attribute specifies the maximum number of characters allowed in an input field. maxlength = "5"
    Max and Min: The input min and max attributes specify the minimum and maximum values for an input field. The min and max attributes work with the following input types: number, range, date, datetime-local, month, time and week. max="2030-12-31" and min="1979-12-31"
    Multiple: The input multiple attribute specifies that the user is allowed to enter more than one value in an input field. The multiple attribute works with the following input types: email, and file. multiple
    Pattern: The input pattern attribute specifies a regular expression that the input field's value is checked against, when the form is submitted. The pattern attribute works with the following input types: text, date, search, url, tel, email, and password. pattern="[A-Za-z]{3}" or pattern=[0]{1}[1-9]{3}-[0-9]{7}
    Number:
    Placeholder: The input placeholder attribute specifies a short hint that describes the expected value of an input field (a sample value or a short description of the expected format). placeholder="example"
    Text:
    Required: The input required attribute specifies that an input field must be filled out before submitting the form. required
